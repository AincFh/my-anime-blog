- 使用 Tailwind 的任意值语法

### 4. 动画和交互

- CSS transition 动画
- 悬停效果

- 页面过渡动画
- 流畅的滚动效果

## 设计原则

### 1. 玻璃态核心样式

```css
/* 基础玻璃态 */
.glass {
  background: rgba(255, 255, 255, 0.1);
  backdrop-filter: blur(16px);
  -webkit-backdrop-filter: blur(16px);
  border: 1px solid rgba(255, 255, 255, 0.2);
}

/* 增强玻璃态 */
.glass-strong {
  background: rgba(255, 255, 255, 0.15);
  backdrop-filter: blur(24px);
  border: 1px solid rgba(255, 255, 255, 0.3);
}

/* 阴影效果 */
.shadow-glass {
  box-shadow: 0 8px 32px rgba(0, 0, 0, 0.3);
}
```

### 2. 颜色系统

```css
/* 项目标准颜色 */
:root {
  --aurora-cyan: #00d4ff;
  --aurora-purple: #7b2ff7;
  --aurora-pink: #f72fff;
  --neon-glow: rgba(0, 212, 255, 0.5);
}
```

### 3. 动画原则

- 过渡时长：200-300ms
- 缓动函数：ease-out
- 悬停放大：scale(1.02)
- 点击缩小：scale(0.98)

## 工作流程

### 1. 需求分析

- 理解组件的功能需求
- 确定交互方式
- 考虑响应式需求
- 检查现有组件库

### 2. 设计方案

- 选择合适的玻璃态变体
- 设计颜色渐变方案
- 规划动画效果
- 考虑暗色主题适配

### 3. 实现代码

- 编写 TypeScript + React 代码
- 使用 Tailwind CSS 类
- 添加 TypeScript 类型
- 实现动画效果

### 4. 代码审查

- 检查视觉效果
- 测试交互体验
- 验证响应式布局
- 确保无障碍访问

## 响应格式

当被调用时：

1. 首先阅读并理解需求
2. 提供设计方案说明

## 注意事项

- 不要创建过于复杂的效果
- 保持动画流畅（60fps）
- 确保文本对比度
- 考虑性能影响
- 优先复用现有组件
